# Front-End Deploy Master 项目介绍

## 项目定位

Front-End Deploy Master 是一个面向 Windows 的桌面效率工具，核心目标是把前端项目的日常管理、命令执行、环境变量维护和发布打包流程集中到一个统一入口中。

它适合这类团队或个人场景：

- 本地维护多个前端项目，需要统一管理
- 经常执行 `dev`、`build`、`preview`、`test` 等命令
- 需要频繁导出构建产物并打 zip 发布包
- 希望减少命令行切换、目录切换和手工压缩步骤

## 产品总览

![应用总览](./images/overview.png)

顶部区域提供全局状态与快捷操作，包括：

- 项目总数
- 运行中任务数
- 当前设备 CPU / 内存使用率
- 主题切换
- 添加项目
- 项目列表显隐
- 排序模式切换

左侧用于项目管理，右侧用于任务日志与执行结果查看，整体是典型的“项目操作区 + 控制台区”双栏工作流。

## 核心模块

### 1. 项目管理面板

![项目详情](./images/project-detail.png)

每个项目卡片支持：

- 自动读取项目名称、版本号、包管理器
- 设置项目别名
- 配置 Build 命令
- 配置产物目录、导出目录
- 查看最近一次导出记录
- 一键打开项目目录
- 一键打开最近导出的压缩包

项目详情默认收起，展开后再显示完整配置区域，兼顾列表浏览效率和编辑深度。

### 2. 快捷指令执行

在项目卡片内可以维护 2 到 3 个高频快捷指令，来源支持两种方式：

- 手动填写自定义命令
- 从项目 `package.json` 的 `scripts` 中直接导入

典型命令示例：

- `pnpm dev`
- `pnpm build`
- `npm run dev`
- `yarn preview`

执行后，日志会实时输出到右侧任务控制台，方便观察命令状态和错误信息。

### 3. 一键构建与发布压缩包

发布流程以“打包并导出”为中心，完整链路如下：

1. 执行项目当前配置的构建命令
2. 等待产物目录出现
3. 将产物目录压缩为 zip
4. 输出到导出目录
5. 弹出通知并自动打开文件位置

压缩包命名规则：

```text
项目别名或项目名_YYYY年MM月DD日_HH时mm分ss秒.zip
```

默认导出目录为：

```text
dist-releases
```

这个默认值可以避免把 zip 再次压回 `dist` 内部导致输入输出重叠。

### 4. 环境变量编辑器

![环境变量编辑器](./images/env-editor.png)

环境变量模块支持：

- 自动读取项目根目录下全部 `.env*` 文件
- 在单个界面内切换不同环境文件
- 直接保存当前文件
- 快速创建默认 `.env`
- `Ctrl + /` 注释与取消注释
- 注释行高亮显示

这让“改配置再构建”这类动作不需要频繁切回编辑器。

### 5. 紧凑项目栏

![缩略项目栏](./images/compact-rail.png)

当左侧完整项目列表收起后，应用不会完全隐藏项目入口，而是保留一个更窄的缩略项目栏，用于：

- 保留项目快速切换入口
- 展示项目简称和基本状态
- 在小屏幕或偏控制台操作时节省横向空间

## 典型工作流

```mermaid
flowchart LR
  A["添加项目"] --> B["读取 package.json"]
  B --> C["配置构建命令 / 目录 / 快捷指令"]
  C --> D["运行快捷命令或编辑 .env"]
  D --> E["点击打包并导出"]
  E --> F["生成 zip 发布包"]
  F --> G["通知完成并打开目录"]
```

## 价值点

- 把前端项目管理、命令执行和发布打包整合到一个桌面工具中
- 降低频繁切换终端、文件夹、压缩软件带来的操作成本
- 让重复发版动作从“多步手工操作”变成“一键完成”
- 对多项目并行维护场景更友好
