# Front-End Deploy Master 使用教程

本文档按“从安装到打包发布”的顺序说明主要使用流程。

## 1. 安装与启动

在项目根目录执行：

```bash
npm install
npm run dev
```

如果需要构建安装包：

```bash
npm run dist:win
```

应用启动后，主界面大致如下：

![应用总览](./images/overview.png)

## 2. 添加前端项目

1. 点击顶部 `添加项目`
2. 在 Windows 文件夹选择器中选择你的前端项目目录
3. 目录内需要存在 `package.json`

应用会自动读取：

- 项目名
- 版本号
- 包管理器
- `scripts`

适合直接添加的典型项目包括：

- Vite
- Vue CLI
- React
- Next.js 静态导出项目
- 其他包含标准 `package.json scripts` 的前端工程

## 3. 配置项目基础信息

点击项目卡片右上角的“展开详情”图标后，可进入完整配置区域。

![项目详情](./images/project-detail.png)

你可以修改：

- 项目别名
- Build 命令
- 产物目录
- 导出目录

说明：

- 项目别名优先用于压缩包命名
- 默认产物目录通常是 `dist`
- 默认导出目录为 `dist-releases`

推荐做法：

- 如果你的构建产物在 `build` 目录，把“产物目录”改成 `build`
- 不建议把导出目录放在 `dist/releases` 这种产物目录内部

## 4. 配置并执行快捷指令

项目详情中的“快捷指令”区块支持两种配置方式：

- 直接手动添加自定义命令
- 从 `package.json scripts` 中选择并导入

常见示例：

- `pnpm dev`
- `pnpm build`
- `npm run preview`
- `yarn test`

配置完成后，点击项目卡片上的快捷按钮即可执行。

## 5. 查看任务控制台

命令执行后，右侧“任务控制台”会实时显示：

- 当前任务状态
- 任务开始 / 结束时间
- 错误信息
- 标准输出和错误输出日志

如果某个任务正在运行，还可以点击停止图标主动终止。

你也可以：

- 切换任务列表显示 / 隐藏
- 查看已完成任务的日志
- 打开最新输出文件位置

## 6. 一键构建并导出发布包

点击项目卡片上的 `打包并导出` 按钮后，应用会自动执行完整发布流程：

1. 运行当前 Build 命令
2. 等待产物目录生成
3. 压缩产物目录
4. 输出 zip 到导出目录
5. 弹出系统通知
6. 自动打开压缩包所在目录

压缩包命名格式：

```text
项目别名或项目名_YYYY年MM月DD日_HH时mm分ss秒.zip
```

示例：

```text
ADMIN_2026年04月07日_21时30分45秒.zip
```

## 7. 编辑 `.env` 环境变量

在项目详情下方可以展开“环境变量”区块：

![环境变量编辑器](./images/env-editor.png)

支持的操作：

- 自动列出项目根目录下所有 `.env*` 文件
- 点击左侧文件列表切换编辑对象
- 直接编辑并保存内容
- 新建默认 `.env`
- 注释行高亮显示

快捷键：

```text
Ctrl + /
```

作用：

- 注释当前行
- 取消注释当前行
- 对多行选区批量注释 / 取消注释

## 8. 使用缩略项目栏

如果你希望把更多空间留给任务控制台，可以隐藏左侧完整项目列表。  
隐藏后不会完全失去项目入口，而是保留缩略项目栏：

![缩略项目栏](./images/compact-rail.png)

适合场景：

- 同时关注任务日志
- 屏幕宽度有限
- 只需要快速切换项目

## 9. 项目排序

如果你想调整项目展示顺序：

1. 点击顶部排序模式图标
2. 在左侧项目列表中拖动项目卡片
3. 释放后顺序会自动保存

下次启动应用时，会保留上一次排序结果。

## 10. 顶部系统状态信息

顶部会显示当前设备的：

- CPU 使用率
- 内存使用率

这部分信息由 Electron 主进程轻量采样，不依赖额外大型监控组件，主要用于在发版和构建时快速观察本机负载状态。

## 常见问题

### 修改了主进程代码，为什么界面没更新？

如果改动的是下面这些文件，需要完整重启：

- `electron/main.cjs`
- `electron/preload.cjs`

重新执行：

```bash
Ctrl + C
npm run dev
```

### 为什么导出目录不建议放进 `dist/releases`？

如果压缩源目录就是 `dist`，而导出目录又在 `dist` 内部，压缩输入和输出会发生重叠，可能导致压缩异常或卡住。  
推荐继续使用默认的 `dist-releases`。

### 如何重新生成文档截图？

在项目根目录执行：

```bash
npm run docs:screenshots
```

该命令会先构建前端产物，再用 Electron 截图脚本重新生成 `docs/images/` 内的文档图片。
